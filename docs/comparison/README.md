# Same prompt, with and without solidsight

**Prompt (identical for both sides):** *"make a detailed 3D model of an
inline-4 engine block"*.

The only variable is tooling: no feedback loop vs. the solidsight loop.
Nothing on the blind side was touched after its single run, and both
results are audited by the same validator and rendered by the same
renderer.

**Methodology note (transparency).** A first pilot of this comparison was
discarded: its "blind" script was written by an author who had already
built [example 07](../../skill/examples/07-engine-block) in the same
session, and the result inherited its datums — too good to be honest. The
baseline below was regenerated by a **fresh-context agent** (no knowledge
of this repository, the session, or example 07) under hard constraints:
numpy + trimesh only, no CAD tools or skills even if installed, no
visualization of any kind, one design pass, crash-fix reruns only. It
finished in one pass with zero retries, convinced of success:
`is_watertight: True`.

| | without (blind one-shot) | with (skill + loop) |
|---|---|---|
| environment | plain Python + trimesh, no renders, no report, no catalog | solidsight + SKILL.md |
| process | one pass, shipped on "exported, watertight=True" | feature spec, region-by-region, build+inspect each region |
| iterations | 1 (by definition — nothing to look at) | 4 (each guided by a finding) |
| artifact | [`blind/engine_blind.py`](blind/engine_blind.py) -> [`engine_blind.stl`](blind/engine_blind.stl) | [`skill/examples/07-engine-block`](../../skill/examples/07-engine-block) |

## What the audit found in the blind output

Credit first: the blind block is genuinely competent. Watertight, a single
shell, no sealed cavities, correct siamesed liners on 93 mm spacing,
head-bolt columns rising through a closed-deck jacket, five main bearing
bulkheads with cap pockets, freeze plugs, and an oil-filter-to-gallery
drilling that the audit **exonerated** — it meets the gallery exactly where
intended ([`queries.txt`](blind/out/queries.txt)).

But three of its internal oil/structure features are silently wrong, and
all three are invisible without renders, reports or queries:

| defect | evidence (verbatim tool output in [`queries.txt`](blind/out/queries.txt)) |
|---|---|
| the **main oil gallery is tangent to the block's outer face**: a 0.45 mm skin over a full-length pressurized oil line (drilled at y=-67 r8; face at y=-75) | print-safe audit: FAIL `min wall 0.439 mm at (273.8, -75, 92.4)`; ray at that point: `thickness 0.4523 mm` then the 15 mm gallery void |
| both **head oil drain-backs dead-end inside the wall**: they run from the deck down to z=68 and stop in solid material — they never reach the crankcase they exist to drain into — while passing **1.0 mm from the water jacket** for 100 mm | ray at (90, ·, 160): `1.0 mm` skin between drain and jacket; section y=66: two full-height buried channels (x 84–95, 325–336), no exit at the bottom |
| the **crankcase is open to the outside**: a 5 mm horizontal slot (z 80–85) rings all four bays where the wide skirt meets the narrower bank | `ray 160 -200 82.5 0 1 0` -> **crossings: 0** — a ray passes through the whole engine without touching material; the slots are visible as light through the skirt in `01_iso.png` |

The failure mode is precise: every defect is a **relationship between two
features** (gallery vs. outer face, drain vs. jacket, bay vs. skirt
height). Blind spatial reasoning holds each feature correct in isolation
and misses the interactions — and `watertight=True` says nothing about any
of this.

## The with-tool side made the SAME kind of mistakes

That is the honest core of this comparison. Building the with-tool block
(example 07), the author's errors were equivalent — the loop just caught
each one:

1. Coolant-port cutters missed the block entirely -> `noop-difference`
   warning with both bounding boxes.
2. Both engine mounts contained **sealed cavities** (the inclined tapped
   holes were buried under the surface) -> `internal-cavity` FAIL.
3. Jacket-to-bore wall of 0.75 mm -> `thin-wall` finding; the jacket was
   rebuilt as a stadium annulus (which is also how real siamese-bore
   blocks work).
4. Head-bolt drill points broke into the cam tunnel — found by the same
   `query ray` used above, fixed to a measured 5.0 mm wall.

Same kinds of blunders — feature-vs-feature interactions. **The tool is
the difference between shipping them and catching them.**

## Reproduce

```bash
cd docs/comparison/blind
solidsight build audit.py --views iso,iso_back --slice x=163.5 --slice z=160
solidsight build audit.py --print-safe --out out_ps --views iso
solidsight query audit.py ray 273.8 -200 92.4 0 1 0    # the gallery skin
solidsight query audit.py ray 160 -200 82.5 0 1 0      # through the slot: 0 crossings
solidsight query audit.py section y=66 --res 6         # the buried drain channels
```
